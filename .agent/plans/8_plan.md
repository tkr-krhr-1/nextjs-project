# 実装計画書: 🎯 記事作成機能の実装 (#8)

## 1. 概要

本タスクでは、ユーザーがブラウザから記事を作成、プレビュー、および保存できる機能を実装します。これはブログシステムのコア機能の一つであり、複数のサブタスクに分かれています。

## 2. 現状の分析

- `src/app/admin/new/page.tsx` に基本的なフォームが存在する。
- `src/lib/actions.ts` に `createPost` アクションが存在し、ファイルシステムへの保存ロジックがある。
- バリデーションやプレビュー機能が簡易的であり、改善の余地がある。

## 3. 実装フェーズ

### フェーズ 1: 基盤強化 (#9)

- Zod による堅牢なバリデーションの導入。
- エラーハンドリングの改善と日本語化。
- `react-markdown` によるプレビューの高度化。

### フェーズ 2: 機能拡張

- **メタデータ管理**: タグやカテゴリ、公開状態の管理。
- **画像アップロード**: 記事内への画像挿入機能。
- **保存ロジックの強化**: 下書き保存機能や自動保存の検討。

### フェーズ 3: 品質保証とドキュメント

- 結合テストの実施（投稿から表示まで）。
- README および要件定義書の更新。

## 4. 完了定義 (DoD)

1. 全てのサブタスクが完了し、コードがマージされている。
2. 記事の投稿から表示までの一連のフローが正常に動作する。
3. 実装とドキュメントに乖離がない。
